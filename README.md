# 2025 개인역량 강화 프로젝트

### 목표
- **DevOps 실습을 위한 개인 프로젝트**
- 최종 목표: **Docker**, **Kubernetes**, **Public Cloud** 적용

---

### 패키지별 역할 및 기술 스택

| 패키지         | 역할                                                         | 기술 스택                   |
|----------------|--------------------------------------------------------------|-----------------------------|
| `back`         | - REST API 제공<br>- 프론트엔드의 요청에 응답<br>- 데이터 처리 및 관리 | Spring Boot, Java          |
| `front`        | - 클라이언트 요청에 따른 화면 구현<br>- 정적 페이지 렌더링을 위한 SPA | React, Nginx               |
| `data-base`    | - 데이터 저장 및 관리<br>- 관계형 데이터베이스 관리           | PostgreSQL                 |
| `nginx`        | - 리버스 프록시<br>- 프론트엔드 정적 파일 서빙<br>- 백엔드 API 요청 처리 | Nginx                      |
| `docker-compose.yml` | - 여러 컨테이너를 한번에 관리하고 실행하는 설정 파일          | Docker Compose              |
| `README.md`    | - 프로젝트 개요 및 설정 안내 문서                              | 마크다운 (Markdown)        |

---

### 상세 설명

#### 1. Backend (`back`)

- **역할**:
  - **REST API 제공**: 프론트엔드(React)가 요청한 데이터를 처리하고, 응답을 반환하는 역할을 합니다.
  - **데이터 관리**: 데이터베이스(PostgreSQL 등)와 연동하여 데이터를 처리하고, 저장된 데이터를 프론트엔드로 제공합니다.

- **기술 스택**:
  - **Spring Boot**: RESTful API 개발을 위한 Java 기반 프레임워크
  - **Java**: Spring Boot에서 REST API를 구현하는 데 사용되는 언어
  - **MyBatis**: SQL 매핑 프레임워크로, SQL을 직접 작성하여 데이터베이스와의 연동을 처리합니다.
  - **PostgreSQL**: 데이터 저장을 위한 관계형 데이터베이스

---

#### 2. Frontend (`front`)

- **역할**:
  - **클라이언트 요청에 따른 화면 구현**: React를 사용하여 동적인 사용자 인터페이스를 구현합니다.
  - **정적 페이지 렌더링**: `SPA(Single Page Application)` 방식으로, 서버 사이드에서 렌더링되는 HTML 페이지 없이 JavaScript와 API를 활용하여 클라이언트에서 모든 페이지 전환이 이루어집니다.

- **기술 스택**:
  - **React**: 프론트엔드 애플리케이션의 주요 라이브러리로, 컴포넌트 기반의 UI 구현
  - **Nginx**: 정적 파일 서빙 및 API 요청을 백엔드로 리버스 프록시 처리

---

#### 3. Database (`data-base`)

- **역할**:
  - **데이터 저장 및 관리**: 모든 데이터(사용자, 게시글, 댓글 등)를 저장하고 관리하는 역할을 합니다.

- **기술 스택**:
  - **PostgreSQL**: 관계형 데이터베이스 관리 시스템(RDBMS)

---

#### 4. Nginx (`nginx`)

- **역할**:
  - **리버스 프록시**: 클라이언트의 요청을 백엔드(Spring Boot)로 전달하는 역할을 합니다.
  - **정적 파일 서빙**: React로 빌드한 정적 파일들을 서빙하여 사용자가 브라우저에서 페이지를 볼 수 있도록 합니다.

- **기술 스택**:
  - **Nginx**: 웹 서버로서 리버스 프록시 및 정적 파일 서빙을 처리합니다.

---

#### 5. Docker Compose (`docker-compose.yml`)

- **역할**:
  - 여러 개의 Docker 컨테이너를 한 번에 실행할 수 있도록 관리하는 설정 파일입니다. 이를 통해 백엔드, 프론트엔드, 데이터베이스, Nginx 등 모든 서비스를 통합하고 관리할 수 있습니다.

- **기술 스택**:
  - **Docker Compose**: 여러 Docker 컨테이너를 정의하고 실행하는 도구입니다.

---

#### 6. README.md

- **역할**:
  - 프로젝트 개요 및 사용 방법, 환경 설정 등을 설명하는 문서입니다. 개발자가 프로젝트를 이해하고 설정하는 데 도움을 줍니다.

---

### DevOps 적용 목표

1. **Docker**:
   - 각 서비스(백엔드, 프론트엔드, 데이터베이스)를 **Docker 컨테이너**로 패키징하여, **개발, 테스트, 배포**의 일관성 있는 환경을 제공합니다.
   - Docker Compose를 사용하여 여러 컨테이너를 한 번에 실행할 수 있게 설정합니다.

2. **Kubernetes**:
   - Docker 컨테이너를 **Kubernetes 클러스터**에 배포하여, **자동화된 확장성**, **서비스 발견**, **로드 밸런싱** 등의 기능을 제공합니다.
   - 클러스터 내 여러 인스턴스를 운영하면서 **고가용성**을 확보하고, **자동화된 배포**와 **모니터링**을 설정합니다.

3. **Public Cloud**:
   - AWS, Azure, Google Cloud 등 **Public Cloud 환경**에서 애플리케이션을 **배포**하고, **클라우드 서비스**(RDS, S3 등)를 활용하여 서비스를 운영합니다.
   - **CI/CD 파이프라인**을 설정하여 자동화된 배포와 테스트를 구성합니다.

---

### 프로젝트 진행 순서

1. **기능 설계 및 요구사항 분석**:
   - 시스템의 주요 기능(회원가입, 로그인, 게시글 CRUD 등)을 정의합니다.

2. **백엔드 개발**:
   - **Spring Boot**로 REST API를 구현하고, 데이터베이스(PostgreSQL)와 연동합니다.

3. **프론트엔드 개발**:
   - **React**를 사용하여 클라이언트 측에서 사용자 인터페이스를 구현합니다.

4. **Docker 설정**:
   - 각 서비스(백엔드, 프론트엔드, 데이터베이스)를 Docker 컨테이너로 패키징하고, Docker Compose로 모든 서비스를 함께 실행할 수 있도록 설정합니다.

5. **Nginx 설정**:
   - Nginx를 리버스 프록시로 사용하여 프론트엔드와 백엔드 간의 트래픽을 처리하고, 정적 파일을 서빙합니다.

6. **Kubernetes 배포**:
   - Docker 이미지로 컨테이너화된 애플리케이션을 **Kubernetes 클러스터**에 배포하여, 자동화된 배포 및 스케일링을 설정합니다.

7. **Public Cloud 배포**:
   - AWS, Azure, Google Cloud 중 하나를 선택하여, 애플리케이션을 클라우드 환경에 배포하고, **CI/CD 파이프라인**을 설정합니다.

8. **성능 테스트 및 최적화**:
   - 최적화 및 테스트를 진행하여 애플리케이션의 성능을 개선하고, 장애 대응 방안을 설정합니다.

---

### 추가 고려사항

- **보안**: JWT를 사용한 인증 및 권한 관리, HTTPS 적용 등을 통해 보안을 강화합니다.
- **CI/CD**: Jenkins, GitLab CI 등으로 자동화된 빌드, 테스트, 배포 파이프라인을 설정하여 효율적인 개발과 배포 프로세스를 구축합니다.
- **모니터링**: Prometheus, Grafana 등으로 시스템 모니터링 및 로그 수집을 구현하여 장애를 사전에 감지합니다.
